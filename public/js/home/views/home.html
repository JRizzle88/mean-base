Features

Users

Single sign on using passport.js. Includes a function to check for valid sessions.

Extra User Data

Includes extra user data fields such as an admin(true or false), license(string), role(string).

Account and Admin Areas

Both Account and Admin area are specific to the user. If the controller denies the authentication, they are redirected to login.

Partials

Header and Footer partials are included as examples in public/js/partials directory. They are used in the public/js/index.html file.

Conditional Menu Items

Menu items are conditional based on the type of user.

Manageable Structure

The structure is currently set up so each area has it's own set of controllers and views. This could not be the best case in all scenarios.
